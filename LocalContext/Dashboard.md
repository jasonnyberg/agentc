## Handoff Note

**Project**: AgentC/J3 is an internal-alpha persistent cognition runtime moving toward Edict as the authoritative agent control plane over a C++ persistence/transport/concurrency substrate.

**Current State**: G111 is complete; G091 has retained-terminal/drop/abandoned/non-retroactive-cancel lifecycle semantics plus first `yield!`-based cooperative cancellation checkpoints, `worker.edict_lifecycle_status!` abandoned-worker accounting, and documented private arena/slab cleanup boundaries for the current thread-worker backend, G099 has first Edict task/status validators plus async `intern_start!` dispatch enforcement for malformed/over-broad contracts, gather/classify/filter contract examples, explicit result validation for success/evidence fields, confidence/evidence-threshold rejection, and opt-in coordinator-side trust validation, G103 has a first metadata-only static declaration image for `worker.edict` plus logical read-only/static-immortal mounting, G105 has a static slab ownership audit, a first static-immortal no-mutate ownership probe, a tiny allocator-mounted static dictionary lookup probe, exact-mark static leases, logical static mount registry ids, and OS read-only mmap coverage, and G110 now has known-resource abandoned-owner recovery, deterministic lease/expiry recovery plus participant heartbeat renewal and pidfd owner-death recovery for registered resources, logical Root1 resource/lease wrappers in `root1.edict`, non-parking module-backed `root1.await!`, `EdictVM::resume()` for yielded code frames, a first in-process `Root1AwaitScheduler` parking/resume table with logical continuation handles/status, and a documented public parking `await!` contract boundary.

**Next Action**: Continue 🔗[G103](./Knowledge/Goals/G103-BuildTimeStaticCoreDeclarationImageMvp/index.md) / 🔗[G105](./Knowledge/Goals/G105-ReadOnlyStaticSlabOwnershipModel/index.md) by extending registry metadata from process-local mount ids to image ids, manifest hashes, section/root descriptors, and static-slot-table provenance.

**Key Context**:
- Current G110 `root1.await!` is a poll/drain helper only; it returns ready/timeout envelopes and does not park or resume VM continuations.
- Current G110 abandoned recovery has three layers: explicit `recoverAbandonedResource(key, state, owner, reason)`, lease-driven `registerLease` / `renewLease` / `recoverExpiredLeases(nowTick, reason)`, and participant-level `heartbeatParticipant` / `recoverParticipantLeases` used by pidfd owner-death handling; the imported Root1 surface now exposes logical resource handles and lease wrappers, but not raw fds or `ResourceState*` pointers. `EdictVM::resume()` resumes existing yielded code frames; `Root1AwaitScheduler` now exposes logical continuation handles/status and hides VM resumption behind callbacks, but it is still process-local and not durable/session-safe. G091 worker programs can use `yield!` as the first cooperative cancellation checkpoint.
- Current G091 running `worker.edict_drop!` means handle abandonment, not thread preemption; abandoned detached workers unwind naturally and suppress orphan completion publication.
- Current G099 validators remain opt-in for blocking `intern_run!`, while async `intern_start!` enforces the minimal task contract before dispatch. Result trust validation is explicit/opt-in rather than automatically mutating sync envelopes.

**Do NOT**: Do not add new intern-specific VM opcodes, do not expose raw fds as durable Edict state, do not implement continuation-parking `await!` before the lease/stale-owner resource model is clear, and do not pass stateful provider/runtime handles into intern worker context/imports.
