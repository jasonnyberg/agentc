# Knowledge: J2 VM (Virtual Machine)

**ID**: LOCAL:K012
**Category**: [J2 Archive](index.md)
**Tags**: #j2, #vm, #bytecode, #stackless

## Overview
The J2 VM is a minimalist, stackless bytecode interpreter designed to execute the Edict language. It relies on computed `goto`s for an efficient dispatch loop and manages its own execution stack (`VMRES_CODE`) to avoid C recursion.

## Architecture

### `VM_ENV` (Environment)
- **Thread Local**: `__thread VM_ENV *vm_env`.
- **Resources**: Five primary stacks (Listrees):
  1. `VMRES_DICT`: Dictionary stack (scopes).
  2. `VMRES_STACK`: Data stack (values).
  3. `VMRES_FUNC`: Function call stack.
  4. `VMRES_EXCP`: Exception handler stack.
  5. `VMRES_CODE`: Instruction Pointer / Code stack.

### Dispatch Loop (`vm_run`)
- **Computed Gotos**: Uses GCC/Clang `&&label` address-of-label extension.
- **Structure**:
  ```c
  outer_loop:
      vm_reset_ext();
      vm_code_peek();
  inner_loop:
      vmop_call[*(char *)(vm_env->code_ltv->data++)]();
      goto *inner[!vm_env->state];
  ```
- **State Machine**: Driven by `vm_env->state` (`VM_YIELD`, `VM_BYPASS`, etc.).

## Instruction Set
- **Control Flow**: `RESET`, `YIELD`, `THROW`, `CATCH`.
- **Data/Stack**: `PUSHEXT` (Push Literal), `EVAL` (Execute), `REMOVE` (Pop).
- **Dictionary/Vars**: `REF` (Create Reference), `DEREF` (Resolve Reference), `ASSIGN` (Store).
- **Context/Scope**: `CTX_PUSH`, `CTX_POP` (Push/Pop Dictionary Frame).
- **Functions**: `FUN_PUSH`, `FUN_EVAL`, `FUN_POP` (Manage `VMRES_FUNC`).
- **Shuffles**: `S2S`, `D2S`, `E2S`, `F2S`... (Move items between resource stacks).

## Comparison to J3
- **Similarity**: J3 adopts the "Stack of Lists" model for resources (`VMRES_*`).
- **Difference**: J2 uses specific opcodes for stack shuffling; J3 aims for a more reduced instruction set (though currently similar).
- **Implementation**: J2 is purely C-based with manual pointer arithmetic for bytecode decoding.
